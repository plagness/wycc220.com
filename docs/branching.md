# Ветки и релизы

Практические правила дополняются общим регламентом [repository.guidelines.md](./repository.guidelines.md).

В upstream существуют две постоянные ветки:

- `dev` — целевая ветка всех обычных pull request;
- `main` — production, обновляется релизным pull request из `dev`.

Рабочие ветки находятся в fork контрибьютора. Аварийное исправление также сначала проверяется PR; после `main` оно синхронизируется обратно в `dev`.

## Рекомендуемые rulesets GitHub

Для обеих веток:

- запрет удаления и force push;
- только pull request, минимум 1 одобрение и CODEOWNERS review;
- сброс одобрений после новых коммитов;
- обязательное разрешение обсуждений;
- обязательные проверки `quality` и `secret.scan`;
- запрет обхода правил, кроме отдельной аварийной роли с аудитом.

Дополнительно для `main`:

- разрешать merge только из `dev` организационным правилом/проверкой;
- минимум 2 одобрения;
- deployment environment `production` с ручным одобрением;
- linear history и signed commits при доступности для всех мейнтейнеров.
