# Разработка API

## Назначение

API — общественный слой данных платформы. Его могут использовать основной сайт, фанатские игры и независимые приложения, поэтому стабильность контракта важнее удобства одной текущей реализации.

## Домены первой версии

| Префикс        | Ответственность                          | Команда  |
| -------------- | ---------------------------------------- | -------- |
| `/v1/media`    | видео, стримы, источники и теги          | Media    |
| `/v1/games`    | опубликованные манифесты игр             | Games    |
| `/v1/gallery`  | опубликованные арты и авторство          | Gallery  |
| `/v1/activity` | агрегаты и прозрачный рейтинг            | Activity |
| `/health`      | readiness процесса без внутренних данных | Platform |

Identity и moderation не становятся публичными endpoint автоматически. Административный API проектируется отдельно от публичного домена и другого hostname.

## Contract-first

Сначала меняется `packages/contracts`, затем реализация API, затем потребители. DTO не экспортирует ORM-модели. Каждое поле имеет назначение, формат, nullable-семантику и правило удаления.

## Совместимость

Добавление необязательного поля обычно совместимо. Удаление, переименование, смена типа или смысла поля несовместимы и требуют новой версии. Клиент должен игнорировать неизвестные поля, сервер — строго проверять write-запросы.

## Надёжность и злоупотребления

Все list-endpoint используют cursor pagination. Rate limit применяется по IP и, где есть авторизация, субъекту. Тяжёлые ответы кэшируются. Время внешних API не входит в latency публичного запроса: данные поступают через collectors и очередь.

## Наблюдаемость

Логируются request id, route template, status, duration и безопасные технические признаки. Токены, cookies, query с персональными данными, содержимое чатов и полный request body не логируются. Метрики имеют ограниченную cardinality.

## Definition of done

- контракт типизирован и документирован;
- вход и выход валидируются;
- есть тесты ошибок и abuse limits;
- определены cache и pagination;
- нет прямого внешнего API-вызова в request path;
- документация и changelog обновлены;
- изменения данных согласованы с `docs/data.policy.md`.
