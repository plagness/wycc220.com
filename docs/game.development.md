# Разработка фанатских игр

## Модель платформы

Игра является самостоятельной статической web-сборкой. Она не исполняется внутри доверенного backend и не получает cookies основного сайта. Рекомендуемый движок — Babylon.js, однако каталог остаётся engine-agnostic: допустим любой движок, который выдаёт безопасные HTML/CSS/JS/WASM assets.

## Быстрый старт

```bash
pnpm install
pnpm --filter @wycc220/game.starter.babylon dev
```

Шаблон находится в `games/starter.babylon`. Для отдельной большой игры предпочтителен собственный репозиторий команды; в центральный каталог поступают проверенный `game.manifest.json`, сборочный provenance и ссылка на исходники.

## Обязательные файлы

```text
game.manifest.json
index.html
assets...
```

Манифест содержит версию схемы, стабильный slug, название, краткое описание, авторов, исходники, лицензию, launch URL и минимальные permissions. Схема определяется в `packages/contracts`; неизвестные разрешения отклоняются.

## Графика и форматы

- сцены и модели: glTF/GLB;
- текстуры: KTX2/Basis с fallback при необходимости;
- геометрия: Meshopt или Draco после измерения времени decode;
- звук: сжатые web-форматы и запуск только после действия пользователя;
- большие уровни: динамические chunks и asset streaming;
- качество: профили low/medium/high, а не одна настройка для всех устройств.

## Telegram Mini Apps

Telegram запускает игру во встроенном WebView. WebGL 2 считается базовой целью, WebGPU — улучшением. Игра обязана реагировать на viewport и visibility, поддерживать touch, не требовать hover и сохранять прогресс перед уходом в background. Сервер проверяет Telegram `initData`; клиентским данным доверять нельзя.

## Безопасность песочницы

Production-игра открывается на отдельном origin `*.games.wycc220.com` либо внешнем разрешённом origin. Встраивание выполняется sandboxed iframe с минимальными permissions. Запрещены доступ к parent DOM, попытки получить credentials, произвольная аналитика, реклама, платежи, майнинг, fingerprinting и загрузка исполняемого кода с неизвестных origin.

Обмен с host выполняется через версионируемые сообщения `@wycc220/game.sdk`. Origin и схема сообщения проверяются обеими сторонами.

## Бюджеты производительности

Целевые начальные бюджеты, которые команда может ужесточить после измерений:

- интерактивный старт на среднем мобильном устройстве — до 5 секунд на обычной сети;
- initial transfer без пользовательских уровней — до 8 МБ;
- стабильные 30 FPS на low-профиле;
- отсутствие long task свыше 200 мс во время обычного управления;
- освобождение ресурсов Babylon.js при закрытии/переходе между сценами.

Исключение из бюджета документируется измерением и планом оптимизации.

## Публикация

1. Автор подтверждает права на код и assets.
2. CI собирает игру из зафиксированного commit и формирует checksums.
3. Проверяются manifest, CSP, зависимости, размеры и запрещённые запросы.
4. Модератор проверяет описание, авторство и соответствие хартии.
5. Сборка публикуется на изолированный origin и только затем появляется в каталоге.
6. Новая версия проходит тот же процесс; уже опубликованный bundle не изменяется на месте.

## Definition of done

- управление работает мышью, клавиатурой и touch там, где это уместно;
- есть loading, pause, error и unsupported-device состояния;
- нет секретов и приватных API в bundle;
- лицензии assets перечислены;
- выполнены mobile и Telegram smoke-tests;
- игра не содержит рекламы, донатов и самопродвижения;
- source URL и авторство доступны пользователю.
