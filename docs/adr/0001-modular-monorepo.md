# ADR 0001: модульный монорепозиторий

- Статус: принято
- Дата: 2026-07-15

## Решение

Использовать pnpm workspace и Turborepo. Сайт, API и коллекторы являются отдельными deployable-приложениями; схемы и конфигурация — общими пакетами. Доменные границы поддерживаются внутри приложений без преждевременного выделения микросервисов.

## Причины

Единый репозиторий упрощает атомарные изменения API и UI, единые проверки и вклад новых разработчиков. Раздельные приложения сохраняют независимое развёртывание и изоляцию секретов. Микросервисы на старте увеличили бы операционные риски без измеренной пользы.

## Последствия

Команды обязаны избегать скрытых импортов между приложениями и выносить публичные соглашения в `packages/contracts`. CODEOWNERS и CI могут проверять отдельные области. При росте нагрузки модуль можно вынести без изменения внешнего контракта.
