# Как участвовать

Спасибо за желание помочь. Общий язык проекта и пользовательского интерфейса — русский; имена в коде и публичном API — английские.

## Поток изменений

1. Создайте fork репозитория.
2. Сделайте короткоживущую ветку в своём fork.
3. Не добавляйте ключи, токены, дампы чатов и персональные данные.
4. Выполните `pnpm check`.
5. Откройте pull request в `dev` и заполните шаблон.

Прямые push в `dev` и `main` запрещены правилами GitHub. В `main` попадают только проверенные релизные PR из `dev`.

## Границы проекта

Принимаются улучшения платформы и фанатские работы, связанные с wycc220/elwycco. Не принимаются:

- реклама автора, сервиса, канала, магазина или платного продукта;
- сбор денег, донаты, платные функции, реферальные ссылки и криптовалютные механики;
- трекеры, fingerprinting и сбор данных, не нужных для функций платформы;
- перезалив чужого контента без разрешения;
- код, который загружает секреты в клиент или исполняет непроверенный пользовательский код в доверенном контексте;
- автоматические массовые действия в сторонних сервисах и обход их правил.

Ссылка на автора допустима только как атрибуция конкретной фанатской работы и не должна превращаться в рекламный блок.

## Новая фанатская игра

Игра разрабатывается независимо. В основной репозиторий добавляется небольшой декларативный манифест, а не произвольный серверный код. Публикация проходит модерацию, проверку лицензии, CSP и разрешений. Подробный контракт появится в `packages/contracts` до открытия каталога для публикаций.

## Архитектурные изменения

Изменение формата публичного API, модели идентичности, правил рейтинга, хранения данных или границ безопасности сначала оформляется ADR в `docs/adr/`.
