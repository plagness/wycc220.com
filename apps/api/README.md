# Public API

Backend для `api.wycc220.com`. Он отдаёт версионируемые публичные данные, но не раскрывает credentials коллекторов, внутренние записи модерации и сырые сообщения чатов.

## Структура модуля

```text
src/modules/<domain>/
  routes.ts        HTTP и проверка границы
  service.ts       сценарии предметной области
  repository.ts    интерфейс хранения
  errors.ts        контролируемые ошибки
  *.test.ts        тесты поведения
```

Файлы создаются по мере необходимости. Нельзя помещать SQL в route handler или обращаться из одного домена к таблицам другого домена напрямую.

## Добавление endpoint

1. Определить публичный DTO и схему в `packages/contracts`.
2. Проверить, что endpoint относится к существующему домену.
3. Добавить handler с валидацией входа и сериализацией ответа.
4. Вынести бизнес-правило в service, доступ к данным — в repository.
5. Добавить happy-path, validation, permission и abuse-тесты.
6. Документировать rate limit, пагинацию, ошибки и cache policy.
7. Несовместимое изменение выпускать в следующей версии API.

## Соглашения публичного API

- корень текущей версии — `/v1`;
- коллекции возвращают `{ data, nextCursor }`;
- cursor непрозрачен для клиента;
- даты передаются в UTC ISO 8601;
- идентификаторы являются строками и не раскрывают последовательность базы;
- неизвестное поле входа отклоняется для write-endpoint;
- ошибки имеют стабильные `code`, `message` и `requestId`;
- limit имеет безопасное значение по умолчанию и жёсткий максимум;
- raw chat text не возвращается публичным API.

## Улучшение API

Предложение начинается с issue и примера пользовательского сценария. Изменение контракта сопровождается ADR, миграционным планом и сроком поддержки предыдущей версии. Оптимизация подтверждается измерением: latency, cache hit rate, размер ответа или стоимость запроса.

Подробнее: [`docs/api.development.md`](../../docs/api.development.md).
