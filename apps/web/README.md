# Web application

Публичный сайт `wycc220.com`: каталог медиа, витрина игр, галерея, активность и документация API.

## Границы команды

Web-команда отвечает за маршруты, композицию страниц, доступность, производительность и интеграцию с API. Она не создаёт локальные копии публичных DTO и UI-токенов: контракты импортируются из `@wycc220/contracts`, общие элементы — из `@wycc220/ui`.

## Куда добавлять код

```text
app/                    маршруты и layout Next.js
components/             композиционные компоненты только этого приложения
features/<domain>/      UI и запросы отдельного домена
lib/                    инфраструктурные адаптеры сайта
```

Общий Button, Card, Badge или токен добавляется в `packages/ui`, а компонент конкретной страницы остаётся здесь. Server Component используется по умолчанию; `"use client"` добавляется только для реальной интерактивности.

## Новый раздел

1. Согласовать контракт с API и добавить его в `packages/contracts`.
2. Создать route в `app` и feature-модуль.
3. Использовать существующие UI-примитивы и дизайн-токены.
4. Предусмотреть loading, empty, error и mobile состояния.
5. Добавить доступные названия и управление с клавиатуры.
6. Не обращаться к базе и секретам из web-приложения.

Визуальные правила описаны в [`docs/design.system.md`](../../docs/design.system.md).
